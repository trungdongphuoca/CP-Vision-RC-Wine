Semantic IDs

Hierarchical Semantic ID Design

K-Means phân cấp 3 tầng trên không gian TF-IDF/SVD 128 chiều — ánh xạ 130,000 chai rượu vào 4,096 cụm hương vị.

C1 (×16)
→
Nhóm vùng/giống chính e.g. "Bold Reds"
C2 (×256)
→
Nhóm hương vị trung gian e.g. "Napa Cab"
C3 (×4,096)
→
Nhóm chi tiết ~170 chai/cụm
ITEM (unique)
→
Chai cụ thể 100% unique
# Ví dụ Semantic ID ID = "[07-11-03-042]" # C1=07 → Nhóm vùng Bold Reds # C2=11 → Napa Valley style # C3=03 → Cabernet cluster # ITEM=042 → Chai cụ thể #42 # LLM generates: output = "I suggest the [07-11-03-042]. This Napa Cabernet Sauvignon..."

Phân tích Khoa học

Đánh giá theo 3 Góc độ Nghiên cứu Tiên tiến

Phân tích hiện trạng và hướng nâng cấp dựa trên 3 hướng nghiên cứu được đề xuất bởi hội đồng.

💬
1. Conversational RS + Knowledge-based RS
Đã có: Intent detection, conversation history, LLM sommelier role, NL query understanding.

Thiếu: Knowledge Graph (KG), multi-step reasoning, state management giữa các turn.

Hướng: KGTORe / MKGAT — liên kết Wine → Country → Region → Variety → Flavor Profile
KGTORe · MKGAT · CRS
🕸️
2. Item-Item Graph (LATTICE / FREEDOM)
Đã có: LightGCN embeddings (66MB), TF-IDF→SVD projection, hybrid ChromaDB+GNN (60/40 fusion).

Thiếu: Co-purchase graph từ interaction data, modality-aware edge weighting.

Hướng: Sapo data → item-item graph từ co-purchase patterns → LATTICE-style training
LightGCN · LATTICE · FREEDOM
🖼️
3. Multimodal RS (Vision + Text)
Đã có: YOLO label detection, Florence-2 OCR, CLIP embedding trong code, visual catalog matching.

Thiếu: CLIP visual embedding tích hợp vào ranking score (chỉ dùng cho catalog lookup).

Hướng: Align text query ↔ wine label image → find visually-similar alternatives
CLIP · Florence-2 · MRS
✅
Tóm tắt: Dự án đã đặt nền tảng kỹ thuật vững chắc cho cả 3 hướng nghiên cứu. Các module CV (Florence-2 + CLIP), GNN (LightGCN), và LLM (Llama-3 + conversation history) đều đã hiện diện và hoạt động — chỉ cần deepening integration để tiến từ MVP lên state-of-the-art.
